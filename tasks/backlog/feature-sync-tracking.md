# Feature: Kategorie-Tracking synchronisiert mit Arbeitszeit

**Status:** 📋 Backlog  
**Priority:** P0 (wichtig!)  
**Parent:** Feature 003 + 004

---

## Anforderungen
1. Wird auf aktive Kategorie gedrückt → Tracking dieser Kategorie beendet (Pause)
2. Arbeitszeit pausiert → Kategorie-Tracking pausiert automatisch
3. Arbeitszeit fortgesetzt → Zuletzt aktive Kategorie wird fortgesetzt
4. Arbeitstag beendet → Kategorie-Tracking beendet

## Akzeptanzkriterien
- [ ] Kategorie-Tap = Pause
- [ ] Arbeitszeit-Pause = Kategorie-Pause
- [ ] Arbeitszeit-Resume = Kategorie-Resume (letzte aktive)
- [ ] Arbeitszeit-Ende = Kategorie-Ende
- [ ] Zeitberechnung korrekt
